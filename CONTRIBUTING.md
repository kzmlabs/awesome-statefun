# Contributing Guidelines [![Awesome](https://awesome.re/badge.svg)](https://awesome.re)

Please ensure your pull request adheres to the following guidelines:

- Search previous suggestions before making a new one to avoid duplicates.
- Suggested entries should be tested and useful — working code, clear documentation, demonstrated usage.
- Make an individual pull request for each suggestion.
- Use [title-casing](https://titlecaseconverter.com/) for entry names.
- Use the following format for entries: `[Project name](link) - Description.`
- The description should be a single short sentence ending with a period.
- Keep descriptions short and simple, but descriptive.
- Avoid marketing language ("best", "fastest", "most powerful").
- Avoid defensive framing ("actively maintained", "production-ready") — let the link speak for itself.
- Check your spelling and grammar.
- Make sure your text editor is set to remove trailing whitespace.
- The pull request and commit should have a useful title.

## Quality bar

- Project must be publicly available — open-source preferred, closed-source allowed if the docs are public and substantial.
- Entry must be actively useful — last commit or release within roughly 24 months unless the project is officially archived but historically significant.

Thank you for your suggestions!
